Basics

不要一上来就学工具

先分清 AI、大模型、生成式 AI，不要把它们混成一个概念。

理解 Token、上下文窗口和幻觉，才知道模型为什么会“看起来很懂”。

学会判断能力边界，比追逐每个新工具更重要。

基础认知先打底 04 / 08
RAG & Agent

RAG 和 Agent 到底差在哪？

RAG 更像“先查资料再回答”，适合知识库、客服、制度问答。

Agent 更像“围绕目标去做事”，会计划、调用工具、观察结果。

知识库、工具调用、权限和人工确认，是它们能否落地的关键。

先理解场景，再选方案 05 / 08
MCP & Skill

MCP 和 Skill 也要看懂

MCP 是连接 AI 和外部系统的标准方式，帮助 AI 访问工具、资源和提示模板。

Skill 像“给 AI 的任务方法说明书”，把流程、规则和输出标准沉淀下来。

它们会影响未来 Agent 怎么接工具、怎么稳定完成任务。

Agentic Engineering 必备概念 06 / 08
AI Coding

AI Coding 不是自动写代码

它更像一种新的协作方式：描述任务、提供上下文、审查结果。

真正重要的是代码库上下文、测试、Review、权限和回滚。

Agentic Engineering 关注的是：AI 能做事以后，怎样保持可控。
